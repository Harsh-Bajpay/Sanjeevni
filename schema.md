Title: Firestore Database Schema & Rules
Database Type: Firebase Firestore (NoSQL)

Collection 1: inventory
item_id (String): Unique ID
item_name (String): e.g., "Paracetamol 500mg"
current_stock (Number): Current available units
minimum_threshold (Number): Trigger level for re-order alerts
last_updated (Timestamp)

Collection 2: patients
patient_id (String): Unique ID
name (String)
symptoms_raw (String): The original text inputted by the nurse
triage_level (String): "High", "Medium", or "Low" (Generated by Gemini)
status (String): "Waiting", "In-Consultation", "Discharged"
timestamp (Timestamp)

Collection 3: attendance
staff_id (String)
staff_name (String)
role (String): "Doctor" or "Nurse"
status (String): "Present" or "Absent"
check_in_time (Timestamp)

AI Integration Rules:
Do not allow the Gemini AI to directly write to the database to prevent hallucinations. Gemini should only return a structured JSON response (e.g., {"triage_level": "High", "extracted_symptoms": ["fever", "chest pain"]}). The frontend React application must parse this JSON and execute a deterministic Firestore write operation.
